# プロジェクト: doujin_daihon_generate (BL版)

YouTubeショート用「BL寸止め台本」の自動生成プロジェクト。

## モデル運用方針
- **メインモデル**: Claude Opus 4.6（Antigravity経由、Thinking版）
- **フォールバック**: Gemini 3.1 Pro（Opusのクォータ切れ時）
- 両モデルで同じ設定ファイルが動くこと（プレーンMarkdownを使い、特定モデルのみ得意な構文に依存しない）
- このファイルは `CLAUDE.md` / `AGENTS.md` の両名で配置されている。Antigravityは `AGENTS.md`、Claude Codeは `CLAUDE.md` をそれぞれ優先的に読む。

## ディレクトリ構成
- `要件定義.txt` — 台本生成のルール・フロー・Few-Shot例（システムプロンプト本体）
- `Input/Image/` — 今回処理するBL漫画画像（新しい漫画に差し替えて使う）
- `Output/YYYYMMDD/` — その日の出力ルート
- `Output/YYYYMMDD/NN/` — **`Run` 1回ごとに作る個別フォルダ**（`NN` は `01`, `02`, `03`...）
- `Output/YYYYMMDD/NN/台本.txt` — 台本本体
- `Output/YYYYMMDD/NN/台本_Voice.csv` — Vrew音声合成用CSV
- `Output/YYYYMMDD/NN/ocr_text.txt` — OCR抽出済みテキスト
- `Output/YYYYMMDD/NN/_ocr/` — mokuroの生出力退避先

※ **Inputフォルダには `Image/` だけを置く**。mokuroが生成する `Image.mokuro`/`Image.html`/`_ocr/` は毎回 `Output/{YYYYMMDD}/{NN}/_ocr/` に移動する。
※ **同じ日に何度も `Run` するため、台本ごとにフォルダを完全に分ける**。1回のRunで生成する全成果物は同じ `Output/{YYYYMMDD}/{NN}/` 直下に置く。

## 入力画像の前提
- BL漫画の試し読み画像（PNG）。
- 明示的な濡れ場画像は除外して入れること。台本は要件定義.txt Rule2の描写水準（中庸／レベル2、キス・抱擁・首筋等まで）に従う。

## 新しい漫画を処理する場合
1. `Input/Image/` の画像を新しい漫画のPNGに差し替える
2. `Run` を入力する（OCRは自動実行される）

## 時短のヒント（任意）
mokuroのOCR処理は妻のPC（GPU無し）ではボトルネック（数分〜10分以上）。以下の使い分けで体感時間を短縮できる:
- **`Run`（通常）**: OCRから台本まで自動。所要時間 ＝ mokuro処理時間 ＋ モデル生成時間
- **`Run OCR`**: mokuroだけ先に走らせる（台本は作らない）。漫画差し替え後すぐ実行しておけば、別作業をしている間にOCRが進む
- **`Run 続き`**: 既存OCRを再利用して台本だけ生成（mokuro再実行なし、最速）
- **`Run 差し替え`**: 既存OCRを再利用して、最新フォルダの台本＋CSVを上書き書き直し（mokuro再実行なし）

## 「Run」コマンド

ユーザーが **`Run`** とだけ入力したら、以下を自動実行すること。確認は不要、即実行。

### 実行手順

1. **要件定義の読込**: `要件定義.txt` を読み、Step1〜Step3のフローを頭に入れる。
2. **今回のRunフォルダを決定**:
   - `Output/{今日の日付YYYYMMDD}/` 配下にある既存サブフォルダ（`01`, `02`, ...）を `ls` して、**次の番号**（ゼロ埋め2桁）を割り当てる。
   - 例: 既に `01/` `02/` がある → 今回は `03/`。何もなければ `01/`。
   - 以後の保存先は全て `Output/{YYYYMMDD}/{NN}/` 直下に置く（**この `{NN}` を以下「今回フォルダ」と呼ぶ**）。
3. **OCR実行**（毎回必ず実行する。`Run 続き`/`Run 差し替え` 等の派生コマンドではスキップ）:
   - 今回フォルダ内に `_ocr/` を作成: `mkdir -p Output/{YYYYMMDD}/{NN}/_ocr`
   - Bashで以下を実行: `mokuro --disable_confirmation=True {プロジェクト絶対パス}/Input/Image`
     - 例: `mokuro --disable_confirmation=True C:/Users/syout/Documents/04_Youtube/doujin_daihon_generate/Input/Image`
   - 完了すると `Input/Image.mokuro`（JSON）・`Input/Image.html`・`Input/_ocr/` が生成される
   - **生成物を即座に退避**（Inputを汚さないため）:
     ```
     mv Input/Image.mokuro Output/{YYYYMMDD}/{NN}/_ocr/Image.mokuro
     mv Input/Image.html   Output/{YYYYMMDD}/{NN}/_ocr/Image.html
     rm -rf Output/{YYYYMMDD}/{NN}/_ocr/_ocr && mv Input/_ocr Output/{YYYYMMDD}/{NN}/_ocr/_ocr
     ```
   - 以下のPythonスクリプトをBashで実行して `ocr_text.txt` を生成する:
     ```
     python -c "
     import json
     with open('Output/{YYYYMMDD}/{NN}/_ocr/Image.mokuro', encoding='utf-8') as f: data = json.load(f)
     lines = []
     for page in data['pages']:
         lines.append('=== ' + page['img_path'] + ' ===')
         for block in page['blocks']: lines.append(' / '.join(block['lines']))
         lines.append('')
     open('Output/{YYYYMMDD}/{NN}/ocr_text.txt', 'w', encoding='utf-8').write('\n'.join(lines))
     "
     ```
   - 退避後のInputフォルダは `Image/` のみになっていること。そうでなければ残骸を今回フォルダの `_ocr/` へ移動する。
4. **素材の読込**:
   - `Output/{今日の日付YYYYMMDD}/{NN}/ocr_text.txt` を読む（各ページのセリフ・地の文の正確な内容）
   - `Input/Image/` の画像も把握する（カット指示で「P5の◯◯」等の参照に使う）
5. **STORY把握**（ファイル保存不要・頭の中で整理するだけ）:
   - ocr_textから登場人物・時系列・コアギミックを把握する。
   - **この把握を台本生成の唯一の時系列基準とする。ocr_textに無いシーンは台本に入れない。**
6. **要件定義に従って生成**:
   - 要件定義.txt の Step1〜Step3を順に内部処理する。
   - **ジャンル分類は廃止**しているので、試し読みの内容と感情曲線に最も合う Rule4（フック）/ Rule5（寸止め）の型を選ぶ。
   - 21〜25カットを目安にするが、試し読みの分量が少ない場合は**カット数を減らしてよい**（捏造で水増ししない）。
7. **自己チェック**（出力前・ファイル読み込み不要）:
      - [ ] 全テロップに話者タグ（[ナレ][男主][男脇][女脇]）が付いているか
      - [ ] 1カット40字超えていないか（Rule17）
      - [ ] 3〜5文字の超短文ツッコミが直前のセリフ／描写への「反応」として置かれているか（Rule10）
      - [ ] 寸止め直前1〜2カットが感情ピーク／関係性の転換点で締まっているか（Rule18）
      - [ ] フックが完了形（〜すぎた／〜してみた）で言い切っているか（Rule4）
      - [ ] **【最重要】各ナレ・各セリフが ocr_text または画像のどの部分から導出できるか脳内で言えるか**（Rule9・Rule14・Rule15）
      - [ ] 「実は〜のだが」「どうやら〜らしい」を使ったカットがある場合、その内容が ocr_text から直接読み取れるか（Rule14）
8. **出力保存（必ず2ファイルをセットで出力）**:
    - **① 台本本体**: `Output/{今日の日付YYYYMMDD}/{NN}/台本.txt`
      - **必ず今回フォルダ内に `台本.txt` という名前で1本だけ保存**（フォルダがRunごとに分かれているのでナンバリング不要・上書き不要）
      - 冒頭に `# OP: Y / 寸止め: Z` のメタ行を付与（ジャンル行は不要）
      - 本文は `【画像指示：〇〇】` と `[話者タグ] テロップ本文` を交互に繰り返すプレーンテキスト
      - **画像指示の書式（必須）**: `【画像指示：P{ページ番号} - 「{そのコマのセリフ抜粋}」のコマ】`
        - **ページ番号**: ocr_textの `=== ファイル名 ===` 並び順で `P1`〜`PN`。
        - **セリフ抜粋**: そのコマに含まれるセリフを短く引用する。セリフがない場面は状況を一言で（例：「表紙」「胸板のアップ」）。
        - 例: `【画像指示：P5 - 「離せよ」のコマ】`
        - 例: `【画像指示：P8 - 「無理ですよ先輩」のコマ】`
        - 例: `【画像指示：P1 - 表紙】`
    - **② Voice.csv（Vrew音声合成用）**: `Output/{今日の日付YYYYMMDD}/{NN}/台本_Voice.csv`
      - 形式: `話者名,テロップ本文` の2カラム。ヘッダー行なし、BOM付きUTF-8で保存
      - `【画像指示：〇〇】` 行は**含めない**（ナレ／セリフのみ）
      - CTA（固定エンディング）も1行として含める
      - 話者タグ→話者名の変換は下記「話者マッピング」に従う
9. **完了報告**: 保存パス（今回フォルダ＋2ファイル）、カット数、選択したフック型・寸止め型を1〜2文で報告。

### 話者タグ（BL用）
- `[ナレ]` 三人称ナレーション（タメ口の実況視点）
- `[男主]` 男性主人公（多くの場合、視点者＝受け側）
- `[男脇]` もう一人の男性キャラ（攻め側／相手役、または上司・先輩・友人）
- `[女脇]` 女性脇役（後輩・同僚等が登場する場合のみ）

※BL作品なので `[女主]` タグは使わない。

### 話者マッピング（Voice.csv変換用・Vrew/VOICEVOX想定）
| タグ | 話者名（CSV出力値） |
|---|---|
| `[ナレ]` | 青山龍星 |
| `[男主]` | 白上虎太郎 |
| `[男脇]` | 玄野武宏 |
| `[女脇]` | 春日部つむぎ |

※この対応表を変更したい場合はこの表を編集すれば反映される。

### 禁止事項（重要）
- AIの挨拶・「承知しました」等の前置きは一切出力しない
- **ocr_text／画像に根拠が無い情報をナレで補完しない**（要件定義.txt Rule9・Rule14・Rule15）
- 「カット数を稼ぐため」「セリフ比率を上げるため」に台詞や動機を捏造しない
- 同じ入力で複数回 `Run` された場合は、**新しい連番フォルダ（`02/`, `03/`, ...）を作って完全に分けて保存**する。既存フォルダの上書きは禁止。

## その他のコマンド

### `Run OCR`
mokuroだけ走らせて `ocr_text.txt` を作る（台本は作らない）。**時短用途**。
- 上記「Run」の手順1〜3だけ実行する。
- 台本生成は行わない。完了報告は「OCR完了。次に `Run 続き` を実行できます」とだけ返す。

### `Run 続き`
直前に `Run OCR` で作った最新フォルダの `ocr_text.txt` を使って、**OCRをスキップ**して台本＋Voice.csvだけ生成する。
- 「今回フォルダ」は **直前の `Run OCR` で作った最新フォルダ**を再利用（新規連番を作らない）。
- 上記「Run」の手順4以降だけ実行する。

### `Run 差し替え`
**今日の最新フォルダ**（`Output/{今日の日付}/{NN}/` の中で最大のNN）の `台本.txt` ＋ `台本_Voice.csv` を、**OCRをスキップして**同じ入力で書き直して上書きする（フォルダは増やさない）。
- 既存の `ocr_text.txt` を再利用する。
- 上記「Run」の手順4以降だけ実行し、保存先は最新フォルダの `台本.txt` / `台本_Voice.csv` を上書き。

### `Run CSV`
今日の最新フォルダの `台本.txt` から `台本_Voice.csv` だけを再生成（台本本体は触らない）。

### `Run 切り出し`
今日の最新フォルダの `台本.txt` の `【画像指示】` を元に、mokuro座標を使って `Input/Image/` からコマを自動切り出しし、連番PNGで保存する。**Claude API・GPU不使用。**
- 以下のBashコマンドを実行する:
  ```
  python {プロジェクト絶対パス}/crop_images.py Output/{YYYYMMDD}/{NN}
  ```
- 出力先: `Output/{YYYYMMDD}/{NN}/切り出し/01.png`, `02.png`...（カット順の連番）
- セリフが一致するコマを余白付きでクロップ。マッチしない場合はページ全体を保存。
- `【画像指示：最後の誘導用画像】`（CTAカット）はPページ指定がないのでスキップされる（手動で別途用意）。
- 完了後、保存枚数とパスを報告する。
- **事前条件**: `pip install Pillow` が済んでいること（初回のみ）。
